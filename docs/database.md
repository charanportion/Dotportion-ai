# 1. Database Design Principles

For your product, the database must support:

1. **Product knowledge graph**
    
2. **AI embeddings**
    
3. **Feature prioritization**
    
4. **Future integrations**
    

So we structure it around **core entities**:

User  
Project  
Signal  
Cluster  
Problem  
Feature  
Task  
PRD  
Chat

Relationships:

User → Projects  
Project → Signals  
Signals → Clusters  
Clusters → Problems  
Problems → Features  
Features → Tasks  
Features → PRDs

---

# 3. User Model

model User {  
  id        String   @id @default(uuid())  
  email     String   @unique  
  name      String?  
  createdAt DateTime @default(now())  
  
  projects  Project[]  
}

---

# 4. Project Model

Each SaaS product a user analyzes.

model Project {  
  id          String   @id @default(uuid())  
  name        String  
  description String?  
  createdAt   DateTime @default(now())  
  
  userId String  
  user   User   @relation(fields: [userId], references: [id])  
  
  signals  Signal[]  
  clusters Cluster[]  
  problems Problem[]  
  features Feature[]  
}

---

# 5. Signal Model

Raw feedback.

Examples:

support tickets  
customer feedback  
interviews  
reviews

model Signal {  
  id        String   @id @default(uuid())  
  content   String  
  source    String  
  sentiment Float?  
  
  embedding Unsupported("vector")  
  
  createdAt DateTime @default(now())  
  
  projectId String  
  project   Project @relation(fields: [projectId], references: [id])  
  
  clusterId String?  
  cluster   Cluster? @relation(fields: [clusterId], references: [id])  
}

The `embedding` field stores vectors for **pgvector semantic search**.

---

# 6. Cluster Model

Groups similar signals.

Example:

Import problems  
Onboarding confusion  
Export issues

model Cluster {  
  id          String   @id @default(uuid())  
  name        String  
  signalCount Int  
  createdAt   DateTime @default(now())  
  
  projectId String  
  project   Project @relation(fields: [projectId], references: [id])  
  
  signals  Signal[]  
  problems Problem[]  
}

---

# 7. Problem Model

Clusters become **actual product problems**.

Example:

Data import UX confusing

model Problem {  
  id            String   @id @default(uuid())  
  title         String  
  description   String  
  severity      Float  
  evidenceCount Int  
  
  createdAt DateTime @default(now())  
  
  projectId String  
  project   Project @relation(fields: [projectId], references: [id])  
  
  clusterId String  
  cluster   Cluster @relation(fields: [clusterId], references: [id])  
  
  features Feature[]  
}

---

# 8. Feature Model

Solutions generated by the AI.

Example:

Guided CSV Import Wizard

model Feature {  
  id          String   @id @default(uuid())  
  title       String  
  description String  
  
  impactScore     Float  
  confidenceScore Float  
  effortScore     Float  
  priorityScore   Float  
  
  createdAt DateTime @default(now())  
  
  projectId String  
  project   Project @relation(fields: [projectId], references: [id])  
  
  problemId String  
  problem   Problem @relation(fields: [problemId], references: [id])  
  
  tasks Task[]  
  prd   PRD?  
}

---

# 9. Task Model

Engineering tasks generated from PRD.

Example:

Create import wizard UI  
Add validation API

model Task {  
  id          String   @id @default(uuid())  
  title       String  
  description String  
  status      String   @default("pending")  
  
  featureId String  
  feature   Feature @relation(fields: [featureId], references: [id])  
}

---

# 10. PRD Model

Stores generated product documents.

model PRD {  
  id        String   @id @default(uuid())  
  content   String  
  createdAt DateTime @default(now())  
  
  featureId String  @unique  
  feature   Feature @relation(fields: [featureId], references: [id])  
}

---

# 11. Chat Model (For AI Assistant)

For chat history.

model ChatMessage {  
  id        String   @id @default(uuid())  
  role      String  
  content   String  
  createdAt DateTime @default(now())  
  
  projectId String  
  project   Project @relation(fields: [projectId], references: [id])  
}

Add indexes for performance.

Example:

@@index([projectId])  
@@index([clusterId])

# 13. pgvector Setup

In PostgreSQL enable:

CREATE EXTENSION IF NOT EXISTS vector;

Embedding column example:

vector(1536)

Used for:

semantic search  
similar feedback detection  
clustering

# 14. Example Data Flow

User uploads feedback:

"Importing CSV is confusing"

Stored as:

Signal

After clustering:

Cluster → Import problems

AI reasoning:

Problem → Import UX confusing

AI suggestion:

Feature → Guided import wizard

PRD generated:

Tasks created

Everything connected in the database.
